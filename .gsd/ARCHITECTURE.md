## Components

### Web Layer

#### Authentication (`js/auth.js` + `css/auth.css`)
- **Purpose:** Login, signup, forgot password, Google Sign-In
- **Location:** `index.html`, `js/auth.js`, `css/auth.css`
- **Data:** Hardcoded demo accounts + localStorage for dynamic accounts
- **Session:** `localStorage.ebUser` / `sessionStorage.ebUser`

#### Places Data (`js/places.js`)
- **Purpose:** Static data store for all 120 places
- **Location:** `js/places.js` (40KB)
- **Schema:** `{ id, name, category, zone, lat, lng, rating, entryFee, description, image }`
- **Categories:** Parks, Palaces, Temples, Mosques, Lakes, Malls, Museums, Markets, Restaurants, Attractions, Heritage, Landmarks, Day Trip
- **Zones:** Central, South, North, East, West, Day Trip

#### Map (`js/map.js`)
- **Purpose:** Interactive Leaflet map with markers and geolocation
- **Location:** `js/map.js`
- **Dependencies:** Leaflet.js (CDN), OpenStreetMap tiles
- **Exports:** `window.addMarkers()`, `window.calculateDistance()`, `window.mapCenterTo()`

#### Filters & Cards (`js/filters.js`)
- **Purpose:** Search, category/zone filter, sort, and card rendering
- **Location:** `js/filters.js`
- **Dependencies:** `placesData`, `window.addMarkers`, `window.loadAllPlaceImages`, `window.getAIBadgeHTML`

#### Modal (`js/modal.js`)
- **Purpose:** Place details popup with fare estimates, directions, Uber booking
- **Location:** `js/modal.js`, `css/modal.css`
- **External Links:** Google Maps Directions, Uber deep link

#### Image Loading (`js/images.js`)
- **Purpose:** Local image loading with Wikipedia API fallback
- **Location:** `js/images.js`

#### AI Assist (`js/ai.js`)
- **Purpose:** Scoring algorithm for place recommendations
- **Location:** `js/ai.js`
- **Algorithm:** `score = (rating × 40) - (entryFee × 0.5) - (distance × 2)`
- **Tiers:** Excellent (>150), Good (>100), Consider (≤100)

### Android Layer

#### SplashActivity (`SplashActivity.kt`)
- **Purpose:** Branded splash screen, checks SharedPreferences for login state
- **Duration:** 1.8 seconds
- **Routes to:** LoginActivity or MainActivity

#### LoginActivity (`LoginActivity.kt`)
- **Purpose:** Native Material Design 3 login/signup screen
- **Auth logic:** Same hardcoded accounts as web + local account storage via SharedPreferences/JSON
- **Features:** Email validation, password toggle, remember me, forgot password

#### MainActivity (`MainActivity.kt`)
- **Purpose:** Hosts WebView (loads assets/app.html) + native BottomNavigationView
- **JS Injection:** Logout handler injected after page load

#### WebAppInterface (`WebAppInterface.kt`)
- **Purpose:** `@JavascriptInterface` bridge for WebView → Native communication

## Data Flow

1. **User opens app** → SplashActivity checks `SharedPreferences("is_logged_in")`
2. **Not logged in** → LoginActivity validates credentials against hardcoded + local accounts
3. **Logged in** → MainActivity loads `file:///android_asset/app.html` in WebView
4. **WebView loads** → `places.js` provides data → `filters.js` renders cards → `map.js` adds markers
5. **User taps Details** → `modal.js` shows place info with fare estimates
6. **User taps Uber/Directions** → External link opens via `ACTION_VIEW` intent
7. **User taps Logout** → JS calls `Android.logout()` → clears prefs → routes to LoginActivity

## Technical Debt

- [ ] **Duplicate data stores**: `app.js` has 2801 lines with old place data; `js/places.js` is the canonical source — `app.js` can be deleted
- [ ] **No tests**: Zero test coverage across web and Android
- [ ] **Hardcoded credentials**: Demo accounts are hardcoded in both web (`auth.js`) and Android (`LoginActivity.kt`)
- [ ] **Image size**: 90 images totaling 19.3 MB bundled in Android assets — should be compressed or loaded remotely
- [ ] **No HTTPS enforcement**: WebView allows mixed content (`MIXED_CONTENT_ALWAYS_ALLOW`)
